03 — Advanced SQL

Triggers, Constraints & Stored Procedure

Trigger
Set Occupied on Lease Insert
Auto-updates apartment status when a new lease is created
CREATE TRIGGER
  trg_lease_after_insert_set_occupied
AFTER INSERT ON Lease
FOR EACH ROW BEGIN
  UPDATE Apartment
    SET AptOccupation = 'Occupied'
  WHERE BuildingID = NEW.BuildingID
    AND AptNo = NEW.AptNo;
END
Trigger
Set Vacant on Lease Expiry
Marks apartment vacant when a lease end date passes
CREATE TRIGGER
  trg_lease_after_update_set_vacant
AFTER UPDATE ON Lease
FOR EACH ROW BEGIN
  IF NEW.LeaseEndDate < CURDATE()
  THEN
    UPDATE Apartment
      SET AptOccupation = 'Vacant'
    WHERE BuildingID = NEW.BuildingID
      AND AptNo = NEW.AptNo;
  END IF;
END
Procedure
Assign Staff to Request
Auto-assigns least-busy cleaner to a maintenance request
CREATE PROCEDURE
  sp_assign_staff_to_request(
    IN p_mrid INT)
BEGIN
  -- Find building & apt for request
  SELECT BuildingID, AptNo
    INTO v_building, v_aptno
  FROM MaintenanceRequest
  WHERE MRID = p_mrid;
  -- Pick staff with fewest open tasks
  SELECT c.StaffID INTO v_staff
  FROM Cleans c
  LEFT JOIN MaintenanceRequest mr
    ON mr.AssignedStaffID = c.StaffID
    AND mr.MRStatus = 'Pending'
  WHERE c.BuildingID = v_building
  GROUP BY c.StaffID
  ORDER BY COUNT(mr.MRID) ASC
  LIMIT 1;
END
Constraint
CHECK Constraints
Data integrity enforcement across 4 tables
-- Apartment status values
CHECK (AptOccupation IN
  ('Occupied', 'Vacant'))

-- Lease date integrity
CHECK (LeaseEndDate >= LeaseStartDate)

-- Non-negative rent & deposit
CHECK (LeaseMonthlyRent >= 0)
CHECK (LeaseSecurityDeposite >= 0)

-- Inspection status values
CHECK (InspectionStatus IN
  ('Pending', 'Completed'))

-- Maintenance request status
CHECK (MRStatus IN
  ('Pending', 'Completed'))
04 — Extended Schema

New Entities: Service & ParkingPermit

Service
Tracks additional services requested by corporate clients — from IT support to cleaning and security audits. Enables revenue tracking by service category.
CREATE TABLE Service (
  ServiceID   INT PRIMARY KEY,
  CCID        INT NOT NULL,
  ServiceName VARCHAR(50) NOT NULL,
  ServiceDesc VARCHAR(255),
  ServiceCost NUMERIC(10,2),
  ServiceType VARCHAR(30) NOT NULL,
  FOREIGN KEY (CCID)
    REFERENCES CorpClient(CCID)
);

05 — Real World Impact

Why This Database Matters

Property Management Firms
Companies like CBRE and JLL use relational databases structurally identical to this to track thousands of leases, inspections, and maintenance tickets across their portfolios.
Compliance & Safety
Tracking inspection statuses and scheduled follow-up dates is legally required in most jurisdictions. Our Inspection table mirrors this real-world compliance workflow.
Revenue Analytics
The Lease + CorpClient structure enables instant revenue reporting — critical for financial planning. Our queries return $23,100/month in combined lease revenue instantly.
Automated Operations
Our stored procedure mirrors how modern PropTech platforms auto-dispatch maintenance workers, reducing manual assignment overhead and improving response times.
Trigger-Based Automation
Database triggers that auto-update occupancy status on lease changes are foundational to enterprise systems — eliminating human error in status tracking.
Scalable Entity Design
Adding ParkingPermit and Service entities demonstrates how a normalized schema grows with business needs — no restructuring required, just new tables and foreign keys.
